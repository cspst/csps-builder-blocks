# 一鍵部署說明

兩種方式擇一即可。**方案 A 不用安裝任何軟體，推薦先用。**

---

## 方案 A：GitHub Actions 自動發行（免安裝，推薦）

原理：在 repo 裡放一個自動化腳本，之後**只要上傳檔案，release 就會自動建立**，你再也不用手動發行。

### 一次性設定（約 2 分鐘）

1. 到 repo 頁面 → **Add file → Create new file**
2. 檔名輸入：`.github/workflows/auto-release.yml`（輸入 `/` 會自動建立資料夾）
3. 貼上專案裡 `auto-release.yml` 的內容 → **Commit changes**

### 之後的日常流程（就是「一鍵」）

1. 修改程式後，把 `pxt.json` 的 `version` 調高（如 `0.3.0` → `0.4.0`）
2. **Add file → Upload files** 拖入修改過的檔案 → Commit
3. 完成——Actions 會在幾十秒內自動建立 `v0.4.0` release
   （可到 repo 的 **Actions** 分頁看執行狀態，綠色勾勾＝發行成功）

注意：version 沒調高的話，Actions 會偵測到版本已發行而跳過，這是防呆設計。

---

## 方案 B：deploy.bat 本機一鍵部署（進階）

原理：在自己電腦雙擊 `deploy.bat`，自動完成 commit → push → release 三件事，連瀏覽器都不用開。

### 一次性設定（約 10 分鐘）

1. 安裝 [Git for Windows](https://git-scm.com/download/win)（全部用預設選項）
2. 安裝 [GitHub CLI](https://cli.github.com/)（下載 Windows 安裝檔）
3. 開啟「命令提示字元」執行 `gh auth login`
   依提示選：GitHub.com → HTTPS → Login with a web browser → 瀏覽器按授權

### 之後的日常流程

1. 在本機資料夾修改程式、調高 `pxt.json` 版本
2. 雙擊 `deploy.bat`
3. 看到「完成！」即部署成功

---

## 兩方案比較

| | 方案 A（Actions） | 方案 B（deploy.bat） |
|---|---|---|
| 安裝需求 | 無 | Git + GitHub CLI |
| 操作方式 | 網頁拖曳上傳 | 本機雙擊 |
| 適合對象 | 所有人 | 習慣在本機改程式的人 |
| 自動發行 release | ✓ | ✓ |

兩方案可並存：裝了 Actions 之後，就算用網頁手動上傳、或用 deploy.bat 推送，release 都會自動處理。

## 部署後驗收

到 [minecraft.makecode.com](https://minecraft.makecode.com) 開新專案 → 擴充套件 → 貼 `github.com/cspst/csps-builder-blocks`。
提醒：MakeCode 有快取延遲，剛發行的新版可能要等一段時間（最長數小時）才抓得到，屬正常現象。
