# Minecraft 教育版自訂程式積木完全教學指南

以 `csps-builder-blocks` 擴充套件為例，從零開始學會：製作自訂積木 → 發佈到 GitHub → 讓全校學生匯入使用 → 持續擴充新積木。

---

## 一、基本觀念

### 1.1 Code Builder 與 MakeCode

Minecraft 教育版內建程式編輯器 **Code Builder**（遊戲中按 **C** 開啟），使用微軟的 **MakeCode** 平台。學生可以用「積木 (Blocks)」拖拉寫程式，也能切換到 JavaScript 或 Python 檢視同一份程式。

### 1.2 自訂積木的原理

MakeCode 的每一顆積木，背後都是一段加了特殊註解的 JavaScript 函式。只要在函式前面加上 `//%` 開頭的註解，MakeCode 就會自動把它變成工具箱裡的積木：

```typescript
//% color="#e5a50a" icon="\uf1b2" block="金字塔"     ← 建立工具箱分類（顏色、圖示、名稱）
namespace pyramid {

    //% block="在我前方建造金字塔 底邊 %size"        ← 積木上顯示的文字與參數
    //% size.min=3 size.max=63 size.defl=11          ← 參數的範圍與預設值
    export function buildPyramidHere(size: number): void {
        // 程式內容
    }
}
```

重點規則：一個 `namespace` = 工具箱的一個分類；有 `//% block` 的 `export function` = 一顆積木；沒有 `//% block` 的函式不會出現在工具箱，適合當內部模組。

### 1.3 兩種提供給學生的方式

| | 方式一：custom.ts 貼上法 | 方式二：GitHub 擴充套件 |
|---|---|---|
| 適合情境 | 單堂課、快速測試 | 全校長期使用、持續擴充 |
| 積木存在哪 | 附在那一份專案裡 | 像官方套件，任何專案都能加入 |
| 學生取得方式 | 匯入老師的分享連結 | 在「擴充套件」貼 GitHub 網址 |
| 更新方式 | 重新分享新連結 | 老師更新 repo，學生按更新 |

## 二、方式一：custom.ts 貼上法（5 分鐘上手）

1. 進入教育版世界（建議創造模式＋平坦世界），按 **C** 開啟 Code Builder，選 MakeCode 建立新專案
2. 右上角把「積木」切換成 **JavaScript**
3. 展開左下 **檔案總管 (Explorer)** → 點 **＋** → 同意加入 `custom.ts`
4. 把自訂積木程式碼全部貼進 `custom.ts` 分頁
5. 切回「積木」檢視 → 工具箱出現新分類

> 常見錯誤：程式碼貼在 `main.ts` 不會產生積木，一定要貼在 `custom.ts`。

分享給學生：點編輯器上方「**分享**」→ 發佈專案 → 複製連結。學生在 Code Builder 首頁點「**匯入 → 匯入 URL**」貼上，開啟的專案就帶有這些積木。

## 三、方式二：GitHub 擴充套件（正式做法）

### 3.1 註冊 GitHub 帳號

1. 到 [github.com](https://github.com) 點 **Sign up**
2. 用學校信箱註冊，取一個**短而好輸入**的使用者名稱（它會出現在學生要貼的網址裡）
3. 收驗證信完成註冊

### 3.2 建立儲存庫（Repository）

1. 右上角 **＋** → **New repository**
2. Repository name：`csps-builder-blocks`
3. 選 **Public**（必須公開學生才抓得到）
4. 其他不勾選，點 **Create repository**

### 3.3 上傳擴充套件檔案

1. 空 repo 頁面點 **uploading an existing file** 連結
2. 把專案資料夾裡的檔案全選拖進網頁虛線框：

```
pxt.json      ← 套件設定（名稱、版本、檔案清單）※必要
_shared.ts    ← 共用工具程式
pyramid.ts    ← 金字塔系列積木
test.ts       ← 開發測試用（匯入者不會載入）
README.md     ← 套件說明
LICENSE       ← 授權條款
```

3. 訊息欄填 `v0.1.0 金字塔系列` → **Commit changes**

### 3.4 建立發行版（Release）

1. repo 首頁右側 **Releases** → **Create a new release**
2. **Choose a tag** → 輸入 `v0.1.0` → **Create new tag**
3. **Publish release**

用意：MakeCode 優先抓取發行版。之後你改程式改到一半，學生匯入的仍是穩定版，不會拿到半成品。

### 3.5 學生匯入

1. Code Builder 開啟任一專案 → 工具箱最下方「**擴充套件 (Extensions)**」
2. 搜尋列貼上：`github.com/你的帳號/csps-builder-blocks`
3. 點選加入 → 工具箱出現「金字塔」分類

### 3.6 日後更新流程

1. 修改程式：repo 頁面點檔案 → 右上**鉛筆圖示**線上編輯；或「Add file → Upload files」上傳覆蓋
2. 記得把 `pxt.json` 的 `version` 往上加（如 `0.2.0`）
3. 發新 release（tag `v0.2.0`）
4. 學生在「擴充套件」頁面更新（或移除後重新加入）

## 四、金字塔積木使用教學

### 4.1 積木一覽

| 積木 | 參數 | 說明 |
|------|------|------|
| 建造金字塔 | 底邊、材質、空心、樣式、位置 | 完整版，參數全開 |
| 在我前方建造金字塔 | 底邊、材質 | 簡化版，適合低年級 |

### 4.2 課堂範例

聊天指令蓋金字塔：

```typescript
player.onChat("pyramid", function () {
    pyramid.buildPyramidHere(11, Block.Sandstone)
})
```

法老密室（空心＋平滑斜面）：

```typescript
player.onChat("tomb", function () {
    pyramid.buildPyramid(15, Block.Sandstone, true, PyramidStyle.Smooth, pos(2, 0, 2))
})
```

遊戲中按 **T** 開聊天輸入 `pyramid` 或 `tomb` 即可。

### 4.3 參數小知識

- 底邊用**奇數**（11、15、21…）頂端才是一格尖頂；偶數會是 2×2 平台
- 「空心」會保留一格厚外殼與底層地板，內部就是密室
- 「平滑式」用沙岩階梯鋪斜面；若階梯方向相反，把程式中資料值 0/1 與 2/3 對調

## 五、模組化設計與擴充新系列

### 5.1 專案的模組化架構

```
csps-builder-blocks/
├── _shared.ts    ← 共用工具：csps.offset（座標位移）、csps.fillBox（區域填滿）
├── pyramid.ts    ← 一個系列 = 一個檔案 = 一個 namespace = 一個工具箱分類
├── maze.ts       ← （未來）迷宮系列
├── spiral.ts     ← （未來）螺旋塔系列
└── pxt.json      ← 新增檔案要在這裡登記
```

金字塔內部也刻意拆成模組，是給學生看的「模組化」示範：

| 函式 | 職責 |
|------|------|
| `buildPyramid` | 主流程：迴圈逐層呼叫其他模組 |
| `buildLayer` | 蓋單一層實心正方形 |
| `hollowLayer` | 挖空該層內部 |
| `addSmoothEdges` | 外圍鋪階梯斜面 |

### 5.2 新增系列三步驟（以迷宮為例）

**步驟一**：repo 中 **Add file → Create new file**，命名 `maze.ts`，套用骨架：

```typescript
//% color="#2e86c1" icon="\uf047" block="迷宮"
namespace maze {
    //% block="產生迷宮 大小 %size 牆壁 %material=minecraftBlock 位置 %pos=minecraftCreatePosition"
    //% size.min=5 size.max=41 size.defl=15
    export function buildMaze(size: number, material: number, pos: Position): void {
        // 主流程
    }
    // 內部模組寫在這裡（不加 //% block 就不會出現在工具箱）
}
```

**步驟二**：編輯 `pxt.json`，`files` 陣列加入 `"maze.ts"`，`version` 改成 `"0.2.0"`

**步驟三**：發佈新 release `v0.2.0`

### 5.3 系列慣例（保持一致，方便維護）

- 一個系列一個檔案、一個 namespace、一種代表色
- 共用工具一律放 `_shared.ts`（呼叫 `csps.offset`、`csps.fillBox`），不重複實作
- 每顆對外積木附中文 JSDoc 註解，學生拆解程式時看得懂
- 測試指令寫在 `test.ts`，匯入者不會載入

## 六、//% 註解語法速查表

| 語法 | 作用 | 範例 |
|------|------|------|
| `color` | 分類顏色 | `//% color="#e5a50a"` |
| `icon` | 分類圖示（Font Awesome 編碼） | `//% icon="\uf1b2"` |
| `block` | 積木顯示文字，`%參數名` 是參數洞 | `//% block="建造金字塔 底邊 %size"` |
| `參數.defl` | 參數預設值 | `//% size.defl=11` |
| `參數.min / .max` | 數字滑桿範圍 | `//% size.min=3 size.max=63` |
| `%參數=minecraftBlock` | 參數變成方塊選擇器 | `%material=minecraftBlock` |
| `%參數=minecraftCreatePosition` | 參數變成座標積木 | `%pos=minecraftCreatePosition` |
| `weight` | 積木在分類內的排序（大在前） | `//% weight=100` |
| enum + `//% block` | 產生下拉選單 | 見 `PyramidStyle` |

## 七、常見問題（FAQ）

**Q1：貼了程式碼卻沒出現積木？**
程式要貼在 `custom.ts`（或擴充套件的 `.ts` 檔），貼在 `main.ts` 無效；另檢查函式是否有 `export` 和 `//% block`。

**Q2：學生貼 GitHub 網址找不到套件？**
確認 repo 是 **Public**、網址拼字正確（`github.com/帳號/csps-builder-blocks`）、`pxt.json` 存在且 `supportedTargets` 含 `minecraft`。

**Q3：更新了程式，學生卻拿到舊版？**
MakeCode 抓的是最新 release。改完檔案記得發新 release，學生端也要在擴充套件頁按更新。

**Q4：金字塔頂端不是尖的？**
底邊用了偶數。改用奇數（11、15、21…）。

**Q5：平滑式的樓梯方向不對？**
不同版本的階梯資料值定義可能相異。開啟 `pyramid.ts` 的 `addSmoothEdges`，把 0/1 與 2/3 對調即可。

**Q6：想清掉蓋錯的建築？**
聊天輸入 `/fill x1 y1 z1 x2 y2 z2 air`，或換個位置重蓋。

## 八、教學應用建議：使用 → 拆解 → 擴充

1. **使用**（全年段）：學生拉積木、調參數，觀察底邊 11 與 21 的差異——訓練抽象化與參數概念
2. **拆解**（中高年級）：切到 JavaScript 看 `buildPyramid` 如何用迴圈逐層呼叫 `buildLayer`——理解迴圈與函式分工
3. **擴充**（進階／社團）：模仿模組結構加功能，例如幫金字塔加入口通道、內部寶箱——實踐模組化設計

這套「積木本身就是可閱讀的範例程式」的做法，讓同一份教材能服務三種程度的學生。
